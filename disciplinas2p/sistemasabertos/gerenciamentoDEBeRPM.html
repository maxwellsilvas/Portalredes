◉ Gerenciador de pacotes DPKG do Debian e derivados

O dpkg é o gerenciador de pacotes do Debian e seus derivados como Ubuntu e Linux Mint. Através dele é possível instalar, remover e construir pacotes. A extensão utilizada pelos pacotes criados e que serão instalados é .deb.

Sua instalação é relativamente simples, basta digitar o comando dpkg -i nome_do_pacote.deb e ele fará todo o processo automaticamente. Mas é preciso observar que o dpkg não fará a atualização e instalação de dependências automaticamente, diferente do APT (Advanced Packaging Tool).

Por isso é importante saber quais são as dependências que o seu pacote precisa para poder ser instalado corretamente. Para verificar as dependências necessárias digite dpkg -I nome_do_pacote.deb e observe na saída o item Depends.

Também existe o cenário em que o usuário não verificou as dependências antes da instalação e depois de rodar o comando dpkg -i nome_do_pacote.deb irá receber um erro no final do processo, informando que a instalação encontrou problemas por conta das dependências que faltam.

Para solucionar (ou pelo menos tentar) o problema das dependências que faltaram durante a instalação, rode o comando apt install -f. Este comando é a versão resumida do apt –fix-broken install.

Exemplos de uso dos comandos:

• dpkg -i nome_do_pacote.deb – instala o pacote .deb.

• dpkg -r nome_do_pacote – remove o pacote inteiro exceto os arquivos de configuração.

• dpkg -P nome_do_pacote – remove o pacote e apaga todos os arquivos de configuração.

• dpkg -s nome_do_pacote – exibe o status.

Para informações mais completas, consulte toda a documentação do DPKG.



◉ Gerenciador de pacotes RPM

O rpm (recursivo acrônimo para RPM Package Manager) é um gerenciador de pacotes que pode ser usado para construir, instalar, consultar, verificar, atualizar e desinstalar pacotes de software. Um pacote consiste em um arquivo de arquivos e informações sobre o pacote, incluindo nome, versão e descrição.

Os pacotes RPM possuem o seguinte formato:


pacote-versão-release.plataforma.rpm


Por exemplo, gtk+-3.6.0-2.ram1.x86_64.rpm, significa

• nome do arquivo = gtk+;
• versão = 3.6.0;
• release = 2;
• plataforma = x86_64.

O termo “ram1” indica que o pacote foi desenvolvido para a distribuição Linux Ramone.


➤ Algumas opções do comando

Modo de Consulta
rpm -q [opções]

Exemplos:

• -q aplicativo : exibe a versão do pacote que forneceu o aplicativo especificado.
• -qa : exibe a versão de todos os pacotes instalados.
• -qf arquivo : consulta o pacote que contém o arquivo especificado.
• -qi pacote : exibe as informações sobre o pacote, incluindo nome, versão e descrição.
• -ql pacote : exibe a lista dos arquivos do pacote.
• -qlc pacote : exibe a lista apenas dos arquivos de configuração do pacote.
• -qld pacote : exibe a lista apenas dos arquivos de documentação do pacote.
• -qR pacote : exibe a lista dos pacotes dos quais o pacote especificado depende.
• -qs pacote : exibe o estado de cada arquivo do pacote (normal, instalado ou substituído).

Modo de Verificação

rpm -V|–verify [opções]


Exemplos:

• -V pacote : verifica pacote.
• -Va : verifica todos os pacote instalados.
• -Vf arquivo : verifica o pacote que possui o arquivo especificado.
• -Vp pacote : verifica o pacote instalado e o arquivo de pacote RPM.

Outras opções

• -i pacote : instala pacote.
• -e pacote : desinstala pacote.
• -F pacote : atualiza pacote já instalado (compara as versões).
• –version : informa a versão do aplicativo.
• -v : exibe informações mais detalhadas das operações sendo realizadas.
• -vh : exibe o caractere # à medida em que o arquivo é desempacotado e verifica dependências.
• -U pacote : substituir pacote (desinstalar e instalar novamente).


➤ Exemplos

Para instalar e verificar o pacote que contém o compilador gcc, digite rpm -ivh gcc-4.8.0-0.8.fc19.x86_64.rpm

A instalação acima não será feita caso o pacote seja dependente de outro(s) pacote(s). Neste caso, o sistema especifica o nome do pacote que está faltando, e você deve instalar o pacote citado antes de tentar instalar o gcc novamente.

Para desinstalar o compilador gcc, basta digitar

rpm -e gcc-4.8.0-0.8.fc19.x86_64.rpm


➤ Observações

A distribuição Ubuntu tem na sua interface gráfica a Central de Programas do Ubuntu que permite instalar e remover pacotes. Para instalar pacotes no Linux pode-se também usar apt-get, aptitude, dpkg, yum ou snap.
